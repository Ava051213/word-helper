# 单词记忆助手 (Word Reminder) 升级计划书

## 1. 项目背景与现状
目前项目已完成从单体架构向**模块化架构**的转型，具备了稳固的底层数据持久化（SQLite + SQLAlchemy）和可扩展的 GUI 标签页结构。系统核心算法（艾宾浩斯遗忘曲线）已上线，并配备了基础的 API 缓冲机制。

## 2. 升级目标
通过三个阶段的迭代，将本项目从一个“基础工具”升级为“全方位智能学习助手”。

---

## 3. 迭代阶段规划

### 第一阶段：交互体验优化与多媒体增强 (V1.1 - V1.2)
**目标**：提升学习过程中的沉浸感和易用性。
- **TTS 语音朗读**：
  - 集成 `gTTS` 或 `pyttsx3`，支持单词及例句的一键发音。
  - 在复习模式下支持自动发音。
- **视觉数据看板升级**：
  - 使用 `matplotlib` 或 `plotly` 绘制更精细的学习热力图（类似 GitHub Contribution Chart）。
  - 增加“遗忘趋势预警”图表。
- **UI/UX 细节优化**：
  - 增加标签页切换动画。
  - 增加深色/浅色模式的自动跟随系统功能。

### 第二阶段：核心算法进阶与 AI 赋能 (V1.5)
**目标**：通过人工智能和更多元化的算法提高记忆效率。
- **多模式复习**：
  - 增加“听写模式”（输入单词拼写）。
  - 增加“选择模式”（释义多选一）。
- **AI 智能例句生成**：
  - 接入 OpenAI 或本地 LLM API，为每个单词自动生成贴合用户学习水平的例句。
- **OCR 快速识词**：
  - 支持用户截屏或上传图片，自动提取并导入生词。

### 第三阶段：多端联动与数据生态 (V2.0)
**目标**：打破单机限制，实现数据自由流动。
- **云同步支持**：
  - 增加 WebDAV 支持（兼容坚果云等），实现多台 PC 间的数据同步。
- **多用户系统**：
  - 支持同一台机器上不同用户的学习数据隔离。
- **生态集成**：
  - 开发 Anki 导出插件（`.apkg` 格式）。
  - 增加主流在线词典（如欧路、墨墨）的生词本导入功能。

---

## 4. 技术栈更新建议
- **TTS**: `gTTS` (在线) 或 `pyttsx3` (离线)
- **数据可视化**: `matplotlib`, `seaborn`
- **AI 接入**: `openai` SDK 或 `requests` 对接国内大模型
- **OCR**: `pytesseract` 或 `EasyOCR`
- **云同步**: `webdav4` 库

## 5. 风险评估
- **API 成本**: AI 和 翻译 API 的调用成本需通过更强的本地缓存机制来控制。
- **性能挑战**: 随着单词量增加，统计图表的渲染速度可能变慢，需考虑异步计算。
